Demostración: Modales de Autenticación

Sistema de modales reutilizables para iniciar sesión y registrarse

Características

  • Diseño responsive y moderno
  • Validación de formularios en tiempo real
  • Toggle para mostrar/ocultar contraseñas
  • Estados de carga con spinners
  • Manejo de errores con mensajes específicos
  • Cierre con Escape o clic fuera del modal
  • Integración con APIs del backend

Cómo usar en tus páginas

Simplemente incluye el script y llama a las funciones globales:

<!-- Incluir el script -->
<script src="/javascript/auth-modals.js"></script>

<!-- Abrir modal de login -->
<button onclick="openAuthModal('login')">
  Iniciar Sesión
</button>

<!-- Abrir modal de registro -->
<button onclick="openAuthModal('register')">
  Registrarse
</button>

Campos del formulario de registro

  • Nombre completo: Mínimo 3 caracteres
  • Correo electrónico: Formato email válido
  • Contraseña: Mínimo 6 caracteres
  • Confirmar contraseña: Debe coincidir
  • Teléfono: 10 dígitos (formato colombiano)
  • Dirección: Mínimo 5 caracteres

Campos del formulario de login

  • Correo electrónico: Formato email válido
  • Contraseña: Mínimo 6 caracteres
  • Recordarme: Opción opcional

Integración con Backend

Los modales se integran automáticamente con las siguientes APIs:

  • POST /api/auth/login - Inicio de sesión
  • POST /api/auth/register - Registro de usuarios

Al iniciar sesión exitosamente, se guardan los siguientes datos en localStorage:

  • authToken - Token de autenticación
  • userName - Nombre del usuario
  • loginTime - Timestamp del login

Páginas donde puedes usarlo

Este sistema de modales puede integrarse en cualquier página del sistema:

  • Página principal (inicio.html)
  • Página de productos (productos.html)
  • CRUD de pedidos (pedidos-crud.html)
  • Auditoría (conexion-auditoria.html)
  • Cualquier página que requiera autenticación
